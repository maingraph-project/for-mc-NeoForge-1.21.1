## v0.3.0

- **New Features**:
    - **Addon Support Architecture**:
        - **Universal Event Context Mechanism**: Introduced `EventContextProvider` and `ContextProviders` registry. Third-party mods can now register custom events and provide their own `Level`/`Player` extraction logic without modifying the core code.
- **Optimizations**:
    - **Architectural Refactoring**:
        - **Event Dispatcher Decoupling**: Completely removed hardcoded `instanceof` checks from `EventDispatcher`. The dispatch logic is now fully driven by metadata and providers, significantly improving extensibility and maintainability.
        - **Registry Robustness**: Enhanced `NodeHelper` with overloaded `registerEvent` methods to support seamless binding of custom context providers during node registration.
- **Fixes**:
    - **Type Safety**: Fixed potential compilation issues related to generic type inference in `EventMetadata` and `NetworkService`.
    - **Initialization Flow**: Ensured `EventDispatcher` and `NodeInitializer` follow a more reliable initialization sequence to prevent registry freezing issues during mod loading.

---

- **新特性**:
    - **附属支持架构**:
        - **通用事件上下文机制**: 引入了 `EventContextProvider` 和 `ContextProviders` 注册表。第三方模组现在可以注册自定义事件，并提供自己的 `Level`/`Player` 提取逻辑，而无需修改核心代码。
- **优化**:
    - **架构重构**:
        - **事件分发器解耦**: 完全移除了 `EventDispatcher` 中硬编码的 `instanceof` 检查。分发逻辑现在完全由元数据和提供者驱动，显著提升了可扩展性和可维护性。
        - **注册表健壮性**: 增强了 `NodeHelper`，增加了重载的 `registerEvent` 方法，支持在节点注册期间无缝绑定自定义上下文提供者。
- **修复**:
    - **类型安全**: 修复了 `EventMetadata` 和 `NetworkService` 中与泛型类型推断相关的潜在编译问题。
    - **初始化流程**: 确保 `EventDispatcher` 和 `NodeInitializer` 遵循更可靠的初始化顺序，以防止模组加载期间的注册表冻结问题。
