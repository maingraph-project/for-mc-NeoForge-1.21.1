## v0.3.0

- **新特性**:
    - **蓝图绑定系统**: 引入蓝图绑定机制，支持将特定蓝图脚本绑定至物品。
    - **蓝图工作台**: 专为蓝图管理设计的图形化界面，支持对手持物品进行蓝图绑定与解绑，并内置蓝图库预览。
    - **附属支持架构**:
        - **通用事件上下文机制**: 引入了 `EventContextProvider` 和 `ContextProviders` 注册表。第三方模组现在可以注册自定义事件，并提供自己的 `Level`/`Player` 提取逻辑，而无需修改核心代码。
    - 变量节点的类型改为 ANY
- **优化**:
    - **架构重构**:
        - **事件分发器解耦**: 完全移除了 `EventDispatcher` 中硬编码的 `instanceof` 检查。分发逻辑现在完全由元数据和提供者驱动，显著提升了可扩展性和可维护性。
        - **注册表健壮性**: 增强了 `NodeHelper`，增加了重载的 `registerEvent` 方法，支持在节点注册期间无缝绑定自定义上下文提供者。
- **修复**:
    - **类型安全**: 修复了 `EventMetadata` 和 `NetworkService` 中与泛型类型推断相关的潜在编译问题。
    - **初始化流程**: 确保 `EventDispatcher` 和 `NodeInitializer` 遵循更可靠的初始化顺序，以防止模组加载期间的注册表冻结问题。
    - **蓝图编辑器**: 修复了撤销/重做功能因组件初始化顺序及引用缓存导致的失效问题，增强了状态回滚时的引用一致性；同时解决了节点重叠时的点击穿透问题，统一了基于 Z-Order 的交互判定逻辑。
    - **蓝图系统**: 修复了蓝图映射（mappings.json）在更新后由于服务端缓存未清理导致无法立即生效的问题，实现了路由变更后的级联缓存刷新机制。

## v0.3.0

- **New Features**:
    - **Blueprint Binding System**: Introduced a binding mechanism allowing specific blueprint scripts to be bound to items.
    - **Blueprint Workbench**: A graphical interface designed for blueprint management, supporting binding and unbinding on handheld items with a built-in blueprint library preview.
    - **Add-on Support Architecture**:
        - **Universal Event Context Mechanism**: Introduced `EventContextProvider` and `ContextProviders` registries. Third-party mods can now register custom events and provide their own `Level`/`Player` extraction logic without modifying core code.
    - Changed Variable node type to ANY.
- **Optimizations**:
    - **Architectural Refactoring**:
        - **Event Dispatcher Decoupling**: Completely removed hardcoded `instanceof` checks in `EventDispatcher`. Dispatching logic is now entirely driven by metadata and providers, significantly improving extensibility and maintainability.
        - **Registry Robustness**: Enhanced `NodeHelper` with overloaded `registerEvent` methods to support seamless binding of custom context providers during node registration.
- **Bug Fixes**:
    - **Type Safety**: Fixed potential compilation issues related to generic type inference in `EventMetadata` and `NetworkService`.
    - **Initialization Flow**: Ensured `EventDispatcher` and `NodeInitializer` follow a more reliable initialization order to prevent registry freezing issues during mod loading.
    - **Blueprint Editor**: Fixed undo/redo functionality failures caused by component initialization order and reference caching; resolved click-through issues when nodes overlap by unifying Z-Order based interaction logic.
    - **Blueprint System**: Fixed an issue where blueprint mappings (mappings.json) failed to take effect immediately after updates due to uncleared server-side caches; implemented a cascading cache refresh mechanism for routing changes.
