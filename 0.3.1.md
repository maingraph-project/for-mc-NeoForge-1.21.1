## v0.3.1

- **新增**:
    - 当服务端蓝图运行报错时，直接打印带有 ID 的错误提示。
    - **蓝图运行日志指令**: 新增 `/mgmc log` 命令，打印前100条日志
    - **Self 节点**: 新增 `Self` 节点，用于在蓝图逻辑中动态获取当前执行上下文的挂载目标（触发实体）。
- **优化**:
    - **修复移动检测 Bug**: 修复了由于 Minecraft 原生 `xo/zo` 字段在服务器端处理玩家移动包时被提前同步，导致无法检测到水平（X/Z轴）位移的问题。
    - **移动检测灵敏度提升**: 将“当玩家移动”节点的位移检测阈值从 0.01 格（平方 0.0001）优化为 0.001 格（平方 1E-6）
    - **并发安全加固**: 重构了 `BlueprintRouter.java` 的路由表更新逻辑。
- **其他**：
    - 现在 /mgmc 命令都统一要求管理权限。

## v0.3.1 (English Translation)

- **New Features**:
    - Directly print error messages with IDs when server-side blueprints encounter errors.
    - **Blueprint Run Log Command**: Added `/mgmc log` command to print the last 100 logs.
    - **Self Node**: Added `Self` node to dynamically retrieve the mounting target (triggering entity) of the current execution context in blueprint logic.
- **Optimizations**:
    - **Fixed Movement Detection Bug**: Fixed an issue where Minecraft's native `xo/zo` fields were synchronized prematurely when the server processed player movement packets, causing horizontal (X/Z axis) displacement to go undetected.
    - **Improved Movement Detection Sensitivity**: Optimized the displacement detection threshold for the "On Player Move" node from 0.01 blocks (squared 0.0001) to 0.001 blocks (squared 1E-6).
    - **Concurrency Security Reinforcement**: Refactored the routing table update logic in `BlueprintRouter.java`.
- **Others**:
    - All `/mgmc` commands now uniformly require administrative permissions.
